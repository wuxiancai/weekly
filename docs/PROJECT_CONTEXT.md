# BTCUSDT 模拟自动交易系统上下文

## 目标

构建一个基于币安 BTCUSDT 永续合约 K 线与实时数据的模拟自动交易系统，通过 Web 页面展示数据、策略信号、回测结果和参数优化结果。

## 默认市场与窗口

- 交易对：`BTCUSDT`
- 市场：Binance USDⓈ-M Futures，BTCUSDT U本位永续合约
- 默认周期：`1w`
- 默认历史窗口：`2019-09-02` 到 `2026-06-29`
- 默认趋势参数：`EMA(15)` + `MA(40)`
- 默认本金：`10000`
- 默认复利：`YES`
- 默认杠杆：`2`
- 默认手续费：`0.0005`
- 默认滑点：`0.0005`
- 保证金和盈亏结算：USDT

## 核心能力

- 从 Binance REST API 拉取 K 线并存储到 SQLite。
- 支持 Binance WebSocket kline stream 的脚本入口，用于实时 K 线写入。
- 计算 EMA、MA、RSI、ATR、MACD、Bollinger、ADX、成交量均线等指标。
- 支持做多与做空。
- 使用 ATR 止损、ATR 止盈、反向信号平仓。
- 回测记录逐笔交易、收益、胜率、最大单笔回撤；周期权益曲线回撤保留为对照字段。
- 回测支持固定本金和复利两种资金模型；默认复利，平仓后的权益作为下一笔交易本金。
- 支持可选杠杆倍率，默认 `2` 倍。
- BTCUSDT 按 U本位永续线性合约计算盈亏：合约数量单位是 BTC，盈亏单位是 USDT。
- 参数搜索反推更高收益率的指标组合。
- Web 页面展示图表、指标、回测、优化结果和交易明细。
- 首页为 BTCUSDT 周线回测；右上角 `ETH 回测` 进入 ETHUSDT 日线回测页面，功能和排版与首页一致。
- BTCUSDT / ETHUSDT 的 `1w`、`1d`、`4h`、`1h`、`15m` 默认参数按交易对和周期独立配置；页面切换周期时自动套用对应默认值，避免不同周期互相影响。
- BTCUSDT 日线默认参数已单独优化为无杠杆、复利、最大单笔回撤 50% 内收益优先组合；不影响已固化的 BTCUSDT 周线默认策略。
- BTCUSDT 与 ETHUSDT 的 `4h` 已启用状态切换策略：趋势市使用 EMA/MA 顺势策略，震荡市使用布林带 + RSI 均值回归，过渡市空仓或少交易；两个交易对的 4h 默认参数相互独立。
- BTCUSDT 与 ETHUSDT 的 `1h` 已按本地 `2019-09-02 -> 2026-06-29` 历史数据独立优化默认参数；两个交易对 1h 默认值相互独立，也不复用 4h 默认参数。
- BTCUSDT 与 ETHUSDT 的 `15m` 已参照 1h 状态切换策略新增独立默认参数；15m 因噪音和手续费敏感，默认使用更强 ADX、成交量与趋势间距过滤，不复用 1h 默认参数。当前 15m 默认值已按 `2026-01-01 -> 2026-06-29` 参数搜索结果更新：BTCUSDT 使用均衡参数 `EMA34/MA89, ADX>=45, SL0.8, TP10/2/24, VOL1.75, GAP0.003`；ETHUSDT 使用收益最高参数 `EMA21/MA60, ADX>=35, RSI45-80, SL0.8, TP10/2/24, VOL1.25, GAP0.003`。
- 已新增实盘模拟 / Paper Trading 运行态：默认一个共享 `1000 USDT` 模拟账户，同时运行 `BTCUSDT / 1w`、`BTCUSDT / 1d`、`BTCUSDT / 4h`、`BTCUSDT / 1h`、`BTCUSDT / 15m`、`ETHUSDT / 1w`、`ETHUSDT / 1d`、`ETHUSDT / 4h`、`ETHUSDT / 1h`、`ETHUSDT / 15m` 独立策略；只使用 Binance USDⓈ-M Futures 行情，不提交真实订单。
- Paper Trading 已增加资金使用率配置：默认 BTCUSDT 使用总资金 `80%`、ETHUSDT 使用 `20%`；每个交易对内部按 `15m=0%`、`1h=30%`、`4h=40%`、`1d=20%`、`1w=10%` 分配。开仓保证金受 `交易对比例 * 周期比例` 的槽位额度限制，已有持仓不被强制调整；修改配置后，空闲额度立即影响后续开仓，已占用额度在平仓释放后按新配置生效。
- 模拟交易后台使用 REST 轮询已收盘 K 线，首次启动只预热指标并定位最新已收盘 K 线，避免把历史信号误当成实时成交；后续每根 K 线只处理一次，状态写入 SQLite。
- Web 新增 `/paper` 状态页，展示模拟账户资金、资金使用率配置、当前持仓、策略处理进度、最近平仓和运行日志；策略状态里的参数摘要支持鼠标悬停查看参数解释和数值影响。
- 根目录 `./start.sh` 是统一运行入口：先停止本项目旧 Web/Paper 进程，再启动 FastAPI 回测系统和 Paper runner；在 Ubuntu/systemd 环境手动执行会安装/更新并重启 `weekly-web`，该服务调用 `./start.sh --foreground` 托管同一套启动逻辑；非 systemd 环境才用 `nohup` 后台运行并写入 `runtime/start.pid` 与 `runtime/logs/start.log`。
- `./start.sh` 的启动顺序必须是：先停止旧 `weekly-web` / 旧 `weekly-paper` systemd 服务和本项目遗留 Python 进程；再检查目标端口是否被其他项目占用，被其他项目占用才顺延；最后通过 systemd 启动本项目 Web 回测和 Paper 模拟交易。
- `./start.sh --foreground` 是 systemd 内部托管模式，Web 和 Paper 输出分别写入 `runtime/logs/web.log` 与 `runtime/logs/paper_runner.log`，避免手动终端出现 `Press CTRL+C to quit` 的前台 uvicorn 误导。
- 项目只保留根目录 `./start.sh` 作为启动入口，不再保留 `scripts/start.sh` 包装脚本，避免用户误执行旧入口；运行态可通过 `/api/system/runtime` 查看当前 PID、cwd、git commit 和 Paper 页面标记。
- Web 默认端口为 `8001`；如果端口被本项目进程占用，`start.sh` 会先终止旧进程并复用该端口；如果被其他应用占用，则自动顺延到下一个可用端口。
- Paper runner 会按策略指标周期自动计算预热 K 线数量，环境变量 `PAPER_WARMUP_CANDLES` 与策略需求取较大值，且不少于 60 根 K 线。

## 重要原则

- 不承诺未来盈利，只提供可复现的历史回测与模拟交易框架。
- 回测必须计入手续费和滑点，避免虚高收益。
- BTCUSDT 必须按 Binance USDⓈ-M / U本位永续口径展示和计算，避免误解为 COIN-M 币本位合约。
- 回测必须规避未来函数：信号只能使用上一根已收盘 K 线，交易只能在下一根 K 线开盘后执行。
- 参数优化必须区分样本内训练与样本外测试，不能只用整段历史最高收益作为策略结论。
- 参数优化结果必须连同最大单笔回撤、交易次数、胜率一起展示，避免只看收益率。
- 真实交易前必须另行接入账户权限、风控审核、异常熔断和人工确认，本项目默认只做模拟。
- 实盘模拟不需要 Binance API Key，不具备真实下单能力；如果以后接入真实交易，必须新增独立权限、风控、强平/保证金模型和人工确认。
- 周线默认策略参数已按最终截图固化，后续不再继续调整默认参数。
- 不同周期策略按独立模块看待；周线、日线、4h、1h、15m 参数或逻辑调整不应隐式改变其他周期默认参数。
