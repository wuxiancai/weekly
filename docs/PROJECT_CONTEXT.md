# BTCUSDT 模拟自动交易系统上下文

## 目标

构建一个基于币安 BTCUSDT 永续合约 K 线与实时数据的模拟自动交易系统，通过 Web 页面展示数据、策略信号、回测结果和参数优化结果。

## 默认市场与窗口

- 交易对：`BTCUSDT`
- 市场：Binance USDⓈ-M Futures，BTCUSDT U本位永续合约
- 默认周期：`1w`
- 默认历史窗口：`2019-09-02` 到 `2026-06-29`
- 默认趋势参数：`EMA(15)` + `MA(40)`
- 默认本金：`10000`
- 默认复利：`YES`
- 默认杠杆：`2`
- 默认手续费：`0.0005`
- 默认滑点：`0.0005`
- 保证金和盈亏结算：USDT

## 核心能力

- 从 Binance REST API 拉取 K 线并存储到 SQLite。
- 支持 Binance WebSocket kline stream 的脚本入口，用于实时 K 线写入。
- 计算 EMA、MA、RSI、ATR、MACD、Bollinger、ADX、成交量均线等指标。
- 支持做多与做空。
- 使用 ATR 止损、ATR 止盈、反向信号平仓。
- 回测记录逐笔交易、收益、胜率、最大回撤。
- 回测支持固定本金和复利两种资金模型；默认复利，平仓后的权益作为下一笔交易本金。
- 支持可选杠杆倍率，默认 `2` 倍。
- BTCUSDT 按 U本位永续线性合约计算盈亏：合约数量单位是 BTC，盈亏单位是 USDT。
- 参数搜索反推更高收益率的指标组合。
- Web 页面展示图表、指标、回测、优化结果和交易明细。
- 首页为 BTCUSDT 周线回测；右上角 `ETH 回测` 进入 ETHUSDT 日线回测页面，功能和排版与首页一致。
- BTCUSDT / ETHUSDT 的 `1w`、`1d` 默认参数按交易对和周期独立配置；页面切换周期时自动套用对应默认值，避免周线和日线互相影响。
- BTCUSDT 日线默认参数已单独优化为无杠杆、复利、回撤 50% 内收益优先组合；不影响已固化的 BTCUSDT 周线默认策略。

## 重要原则

- 不承诺未来盈利，只提供可复现的历史回测与模拟交易框架。
- 回测必须计入手续费和滑点，避免虚高收益。
- BTCUSDT 必须按 Binance USDⓈ-M / U本位永续口径展示和计算，避免误解为 COIN-M 币本位合约。
- 回测必须规避未来函数：信号只能使用上一根已收盘 K 线，交易只能在下一根 K 线开盘后执行。
- 参数优化必须区分样本内训练与样本外测试，不能只用整段历史最高收益作为策略结论。
- 参数优化结果必须连同最大回撤、交易次数、胜率一起展示，避免只看收益率。
- 真实交易前必须另行接入账户权限、风控审核、异常熔断和人工确认，本项目默认只做模拟。
- 周线默认策略参数已按最终截图固化，后续不再继续调整默认参数。
- 不同周期策略按独立模块看待；周线参数或逻辑调整不应隐式改变日线默认参数，反之亦然。
